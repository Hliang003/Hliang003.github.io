Mac 系统使用hexo搭建个人博客（一）

第一步：在github上创建仓库

需要注意仓库名称必须是username.github.io，其中username是github的用户名，也就是github主页地址的后缀。比如我的github地址是https://github.com/Amoschc/，我建的仓库名就是Amoschc.github.io。如图所示：

第二步：安装node.js

从官网下载响应的安装包，下载地址，mac下载pkg文件，按照提示一步一步安装即可。

安装完成后，需要验证是否安装成功，在终端查看node和npm的版本，正确显示则说明安装成功。

第三步：安装hexo

执行 sudo npm install -g hexo-cli，安装hexo出现错误，如下图所示：

上网查阅，可以执行 sudo npm install --unsafe-perm --verbose -g hexo 来重新安装。之后执行hexo version正确输出版本号，说明安装成功。

第四步：创建第一篇博客

1.在用户主目录下创建名称为blog文件夹并进入，然后开启hexo服务，具体命令如下：

mkdir blog#创建blog文件夹
cd blog#进入blog文件夹
hexo init#初始化文件夹

执行hexo init初始化命令后发现出错，按照提示执行npm install命令

再执行npm audit fix和npm audit fix --force，结果如下所示。

2.修改配置文件

在blog文件夹下，执行hexo s开启本地服务，没有报错，此时在浏览器访问http://localhost:4000，可以访问本地的博客系统。此时说明我们的环境已经没有问题。

![image-20190531132603449](/Users/chenghengchao/Library/Application Support/typora-user-images/image-20190531132603449.png)

3.博客关联到github仓库

在blog文件夹下有一个名称为_config.yml的配置文件，修改title信息和deploy信息。其中deploy的repo需要换成你自己的GitHub仓库地址，就是我们开头创建的仓库。

![image-20190531132942107](/Users/chenghengchao/Library/Application Support/typora-user-images/image-20190531132942107.png)

这样就实现了关联。之后在blog文件夹下，执行下面的命令，效果如下图

hexo g #生成静态网页
hexo d #部署到GitHub上

![image-20190531133354080](/Users/chenghengchao/Library/Application Support/typora-user-images/image-20190531133354080.png)

发现出错了，查阅相关资料得知原因是没有安装hexo-deployer-git插件，执行npm install hexo-deployer-git --save进行安装。

之后依次执行以下命令：

hexo clean # 清空之前生成的部署信息
hexo g #生成静态网页文件
hexo d #部署到GitHub上

之后如果没有报错，访问username.github.io就可以看到自己的博客首页了。我这里是amoschc.github.io。

附：可能有用的命令
